Layout

⚠️ This section is a stub and is still being completed. Thanks for your patience — documentation is being added gradually.

GDS Editor follows a logical layout, with each section of the application window dedicated to a specific function. The left side of the window is focused on scene control, while the right side is dedicated to component control. Components can be added and created from the work bar at the bottom of the window, and their interaction with the scene is managed from the top of the window.

1Application SettingsAccess global application settings, configuration options, user status, and the help menu.
2Scene TabsNavigate between scenes and create new scenes.
3Scene PaletteDisplays the list of components and actions associated with the scene and its components.
4Category Tab HolderHolds the list of category tabs and the button to create new category tabs.
5Scene AreaThe main workspace where you focus on building and editing your scene.
6Work BarA toolbar at the bottom of the window providing easy access to tools and sources.
7Source ButtonsQuick links for adding components from different sources, such as local files, the asset stash, or external libraries.
8Tool ButtonsQuick access to GDS Editor’s basic tools for creating and manipulating components.
9Scene & Component ToolsTools for controlling how components interact with the scene, including z-order, opacity, and alignment.
10Component InspectorOpens when clicking on a component, showing that component’s properties for editing.
11General WorkspaceThe area outside the main scene focus, useful for staging and organizing.
